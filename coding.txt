# Coding Round Module – Architecture, Workflow & Logic Design

## Objective

The Coding Round module will be a part of the AI Interview System where candidates can solve coding problems in a real-time coding environment. The system should automatically evaluate candidate performance using AI-powered code analysis, test case execution, and interview analytics.

The module should simulate a real technical interview experience similar to modern coding assessment platforms.

---

# High-Level Architecture

## System Components

### 1. Frontend (Candidate Interface)

Responsible for:

* Coding editor UI
* Problem statement display
* Timer management
* Language selection
* Code execution UI
* Submission handling
* Real-time feedback visualization

---

### 2. Backend API Layer

Responsible for:

* Authentication
* Question management
* Submission APIs
* Result processing
* Secure code execution communication

---

### 3. Code Execution Engine (Sandbox)

Responsible for:

* Running candidate code securely
* Test case execution
* Resource isolation
* Output validation

Recommended:

* Docker-based isolated containers
* Sandboxed execution environment

---

### 4. AI Evaluation Engine

Responsible for:

* Code quality analysis
* Logic evaluation
* Time & space complexity estimation
* AI-generated feedback
* Suggestion generation

---

### 5. Database Layer

Stores:

* Questions
* Candidate submissions
* Test cases
* Scores
* Feedback reports
* Interview history

---

# Coding Round Workflow

## Step 1 – Candidate Starts Coding Round

* Candidate clicks “Coding Round”
* System creates interview session
* Questions are assigned dynamically

---

## Step 2 – Problem Display

The UI should show:

* Problem statement
* Constraints
* Examples
* Input/Output format
* Difficulty level
* Allowed languages

---

## Step 3 – Candidate Writes Code

## Features

* Syntax highlighting
* Auto-completion
* Dark/Light mode support
* Run code functionality
* Auto-save support

---

## Step 4 – Run Code

Candidate clicks “Run Code”

System:

1. Sends code to backend
2. Backend forwards to execution engine
3. Sandbox compiles & runs code
4. Returns output/errors

---

# Code Execution Logic

## Execution Pipeline

Candidate Code
↓
Validation Layer
↓
Docker Sandbox
↓
Compilation
↓
Test Case Execution
↓
Output Comparison
↓
Result Generation
